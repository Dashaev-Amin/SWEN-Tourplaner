# Wireframe Annotations — Tour Search & Filter

Two layout variants for the new Tour Search & Filter screen in TourPlanner. Both wireframes are desktop-sized (1200×800) and use the same colour palette and typography to keep the comparison fair — only the structural decisions differ.

## Variant A — Speed-optimised (Power Users)

Designed for users who already know the data model and want to filter quickly through many tours.

1. **Persistent left sidebar with all filters visible** — No menu nesting; experienced users see every filter option without extra clicks, making repeated searches faster.
2. **Sortable column headers (↕ icon)** — Sorting happens inline without opening any modal or dropdown, ideal for users comparing tours by distance, duration, or rating.
3. **Inline row actions (Edit · Del)** — Common actions are one click away per row, avoiding context menus or selection steps.

## Variant B — Clarity-optimised (New / Casual Users)

Designed for users who are exploring and need clear visual cues to make decisions.

1. **Large prominent search bar as the primary entry point** — Communicates "this is where you start" with a single visual anchor; reduces decision paralysis for first-time users.
2. **Filter chips instead of a full sidebar** — Shows only the most common filters; "More Filters" is opt-in. Less overwhelming for users unfamiliar with the data model.
3. **Card layout with map previews** — Visual scanning instead of column comparison; new users can compare tours at a glance via thumbnail, badge, and rating without learning a table's structure.
